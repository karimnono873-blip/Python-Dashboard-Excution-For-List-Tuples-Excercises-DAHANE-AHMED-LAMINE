Question 1: Multiply List Items

Constraint: Write a Python program that multiplies all the items in a list.

  • Sample list = [2, 3, 6]
  • Expected Result = 36
q1_multiply.py
def multiply_list(items): """Multiplies all numerical integers within a given list.""" # Edge case: If the list is empty, return 0 or handle appropriately if not items: return 0 result = 1 for item in items: result *= item return result # Define the sample list sample_list = [2, 3, 6] # Execute the function and print detailed output print("--- List Multiplication Execution ---") print(f"Processing list: {sample_list}") output = multiply_list(sample_list) print(f"Result = {output}")

Algorithm Analysis

Time: O(N) Space: O(1)

We initialize a result variable at 1 (the multiplicative identity). We then iterate through every item in the list exactly once, multiplying it against the running total. The time complexity scales linearly with the size of the list.

Question 2: Sort List of Tuples

Constraint: Write a Python program to get a list, sorted in increasing order by the last element in each tuple, from a given list of non-empty tuples. (Hint: You can use the sort function).

  • Sample List: [(2, 5), (1, 2), (4, 4), (2, 3), (2, 1)]
  • Expected result: [(2, 1), (1, 2), (2, 3), (4, 4), (2, 5)]
q2_sort_tuples.py
def get_last_element(t): """Helper function that returns the last element of a tuple.""" return t[-1] def sort_by_last_element(tuples_list): """Sorts a list of tuples based on the last element.""" # We could use the helper function above: key=get_last_element # Or we can use an anonymous lambda function for brevity: return sorted(tuples_list, key=lambda x: x[-1]) # Define the sample list sample_list = [(2, 5), (1, 2), (4, 4), (2, 3), (2, 1)] # Execute the function and print result = sort_by_last_element(sample_list) print(f"Original List:\n{sample_list}\n") print(f"Sorted Result (by last element):\n{result}")

Algorithm Analysis

Time: O(N log N) Space: O(N)

Python's built-in sorted() function uses Timsort. By passing a custom key argument utilizing a lambda expression (lambda x: x[-1]), we instruct the sorting algorithm to evaluate each tuple solely by its last index instead of its first.

Question 3: Combine Dictionaries

Constraint: Write a Python program that combines two dictionaries by adding values for common keys.

  • d1 = {'a': 100, 'b': 200, 'c':300}
  • d2 = {'a': 300, 'b': 200, 'd':400}
  • Expected result: {'a': 400, 'b': 400, 'd': 400, 'c': 300}
q3_combine_dicts.py
from collections import Counter def combine_dicts_with_counter(dict1, dict2): """Combines two dicts, adding values of common keys using Counter.""" # Counter naturally adds values for intersecting keys when using '+' combined = Counter(dict1) + Counter(dict2) return dict(combined) def combine_dicts_manual(dict1, dict2): """Combines two dicts using a standard loop and .get() fallback.""" result = dict1.copy() for key, value in dict2.items(): # If key exists, add value. If not, default to 0 and add value. result[key] = result.get(key, 0) + value return result # Define the dictionaries d1 = {'a': 100, 'b': 200, 'c': 300} d2 = {'a': 300, 'b': 200, 'd': 400} print(f"Dict 1: {d1}") print(f"Dict 2: {d2}\n") result_counter = combine_dicts_with_counter(d1, d2) print(f"Combined (Counter approach): {result_counter}")

Algorithm Analysis

Time: O(K) Space: O(K)

Where K is the total number of unique keys across both dictionaries. Using Python's collections.Counter provides a mathematically elegant solution, as it overloads the + operator to automatically aggregate values mapped to matching string keys.

Question 4: Generate Squares Dictionary

Constraint: With a given integral number n, write a program to generate a dictionary that contains (i, i*i) such that `i` is an integral number between 1 and n (both included).

  • Input: 8
  • Expected output: {1: 1, 2: 4, 3: 9, 4: 16, 5: 25, 6: 36, 7: 49, 8: 64}
q4_squares.py
def generate_squares_dict_loop(n): """Generates a dictionary using a standard for loop.""" squares = {} for i in range(1, n + 1): squares[i] = i * i return squares def generate_squares_dict_comp(n): """Generates a dictionary using dict comprehension (Pythonic way).""" # Syntax: {key: value for item in iterable} # range(1, n + 1) ensures both 1 and n are included. return {i: i * i for i in range(1, n + 1)} # Test the function with input n = 8 input_n = 8 result = generate_squares_dict_comp(input_n) print(f"Input supplied: {input_n}") print(f"Generated Dictionary:\n{result}")

Algorithm Analysis

Time: O(N) Space: O(N)

While a standard for loop works perfectly, this code demonstrates Dictionary Comprehension. This is a highly optimized, single-line "Pythonic" approach to constructing dictionaries that runs slightly faster at the interpreter level than appending via a loop.

Question 5: Sort Tuple by Float Element

Constraint: Write a program to sort a tuple by its float element.

  • Input: [('item1', '12.20'), ('item2', '15.10'), ('item3', '24.5')]
  • Expected result: [('item3', '24.5'), ('item2', '15.10'), ('item1', '12.20')]
q5_sort_float.py
def sort_by_float(tuple_list): """ Sorts list of tuples descending based on the float conversion of the string located at the second index. """ # 1. key=lambda x: float(x[1]) parses the string into an actual float # so mathematical sorting occurs rather than alphabetical sorting. # 2. reverse=True triggers descending order to match the expected output. return sorted(tuple_list, key=lambda x: float(x[1]), reverse=True) # Define sample data (Notice the numbers are actually strings) sample_data = [('item1', '12.20'), ('item2', '15.10'), ('item3', '24.5')] # Execute and print result = sort_by_float(sample_data) print(f"Expected descending result:\n{result}")

Algorithm Analysis

Time: O(N log N) Space: O(N)

Because the numbers are enclosed in quotes (e.g., '24.5'), they are string types. If we sorted them without the float() cast in the lambda key, the algorithm would perform lexicographical sorting, which behaves unpredictably with decimal string numbers. By casting to float, we ensure accurate numerical comparison.

Question 6: Set Operations

Constraint: Write a Python program to create a set. Then iterate over the set. Finally, add members to the set and remove items from it.

q6_sets.py
# 1. Create a set using curly braces my_set = {0, 1, 2, 3, 4} print(f"Initial Set (Type: {type(my_set)}): {my_set}\n") # 2. Iterate over the set print("Iterating over set elements:") for item in my_set: print(f" - Element: {item}") # 3. Add members to the set # Note: Adding an existing element (like 1) has no effect as sets enforce uniqueness my_set.add(5) my_set.add(6) my_set.add(1) print(f"\nSet after adding 5, 6 (and attempting to add duplicate 1):\n{my_set}") # 4. Remove items from the set # Using discard() safely removes an item. It does nothing if the item doesn't exist. my_set.discard(2) # remove() raises a KeyError if the item is not found, so it's stricter. my_set.remove(0) print(f"\nFinal Set after removing 0 and 2:\n{my_set}")

Algorithm Analysis

Lookups: O(1) Insertions: O(1)

Sets in Python are implemented as Hash Tables. This makes checking for membership (e.g., if x in my_set:), adding members, and removing members incredibly fast. However, because they are hashed, sets are inherently unordered and do not support index-based retrieval (like my_set[0]).
